1. Está instalado el paquete que me provee de cron?

2. Comprobar que se está ejecutando el demonio cron. Si no lo estuviera , cómo podría iniciarlo?

3. Como un usuario sin privilegios. Programa una tarea que imprima en el terminal pts/0 "son las nueve" todos los días de lunes a viernes a las 21:00. (Podeí­s también usar el comando wall...man wall)

4. Como el anterior pero solo de los meses enero y febrero.

5. Como un usuario diferente de root y student crear una tarea periódica tal que cada dos minutos los miércoles escriba la fecha y hora en la consola virtual 4.

6. Programa una tarea dentro de cinco minutos de la hora y fecha actuales, con crontab que cree un fichero sin contenido llamado file_temp_actualizado en el directorio home de student. Dicha tarea se debe realizar siempre a esta hora de Lunes a Viernes los meses de verano (Junio, Julio, Agosto, Septiembre).

7. Programa una tarea que haga una compresión de la home de root en un archivo llamado backup_root.tar en /tmp todos los viernes a las 17:30h.

8.  Copia las lí­neas que debes poner en tu crontab (como student) para que se ejecute el script
/usr/bin/miscript ...
  *“ Todos los dí­as a las 13:30 y a las 23:30
  *“ De lunes a viernes a las 12:05
  *“ Cada dos horas los días 15 de enero y junio

9. Realizar una tarea programada para que se reinicie el equipo todos los miércoles a las 20:00.


SOLUCION:

1. Paquete que provee cron:

	yum provides crontab => cronie
	yum info cronie => instalado por Anaconda 

2. Ver si se ejecuta demonio del cron:

	rpm -ql cronie | grep service => crond.service
	systemctl status crond => Arrancado y activado, si no lo estuviere: systemctl start crond y systemctl enable crond

3. Imprimir "son las nueve":

	crontab -e => Abrir el editor del crontab
	0 21 * * 1-5 echo "son las nueve" > /dev/pts/0
	(o con wall "son las nueve" => Lo envia a todo el mundo conectado)

4. Variación del anterior
	crontab -e => Abrir el editor del crontab
	0 21 * 1-2 1-5 wall son las nueve
		 NOTA: 1,3 => Enero, marzo     y      1-3 => Enero,Febrero y Marzo

5. Escribir fecha y hora en la consola:

	Logarte a otro usuario diferente a root y student
	echo "*/2 * * * 3 date > /dev/tty4" | crontab
		--> Mejor: con date +%D-%T  pero ojo!! escapar los %

6. Creación del fichero vacío "file_temp_actualizado":

	-- Veo la hora y fecha actual, p.e. 11:17
	crontab -e
		22 11 * 6-9 1-5 touch ~/file_temp_actualizado

    Mas correcto sería, si existe el fichero, borrarlo y crearlo a continuación:

		22 11 * 6-9 1-5 if [ -e ~/file_temp_actualizado ];then rm ~/file_temp_actualizado;fi;touch ~/file_temp_actualizado


7. Compresión de la home de root:

	echo "30 17 * * 5 tar czf /tmp/backup_root.tar /root" | crontab


8. Ejecutar miscript:

	crontab -e
		30 13,23 * * * /usr/bin/miscript
		5 12 * * 1-5 /usr/bin/miscript
		0 */2 15 1,6 * /usr/bin/miscript


9. Tarea del sistema de reiniciar equipo los miércoles:

       echo '0 20 * * 3 root /sbin/reboot' >> /etc/crontab

--> el comando es /sbin/reboot o systemctl reboot
