Crear una ruta estática
-----------------------
Por defecto todo el tráfico de red normalmente se configurará por el gateway por defecto.
Si para una determinada IP o subred no quiero esto, puedo añadir rutas estáticas para forwardear
un  tráfico concreto.

Con ip route veo el enrutamiento actual.


-> En una conexión:
 nmcli con mod <conexión> +ipv4.routes "<destino> <via>"

 donde <destino> el la IP o dir.red a donde queremos llegar
 y <via> es el gateway por donde llegaremos al destino

Esto crea un fichero /etc/sysconfig/network-scripts/route-<conexión> con el
formato: 
ADDRESS0=<dir_subred_destino>
NETMASK0=<mascara_destino>
GATEWAY0=<via>

reiniciar la red con systemctl restart network o
la conexión con nmcli con down <conexión>; nmcli con up <conexión>
o ifdown <interfaz>;ifup <interfaz>

-> Con el comando ip route:
ip route add <destino> via <via> 
 y se puede añadir al final el nombre del interfaz donde queremos 
añadir el nuevo enrutamiento con "dev <interfaz>"

Pero no es persistente!!!! Para hacerlo persistente p.e. en el interfaz eth0:

Abrir/Editar el fichero /etc/sysconfig/network-scripts/route-eth0 
Y poner (p.e.):

GATEWAY0=<via1>
PREFIX0=<prefix-destino1>
ADDRESS0=<dir_subred_destino1>

GATEWAY1=<via2>
PREFIX1= <prefix-destino2>
ADDRESS1=<dir_subred_destino2>
